  ✓ Features:
    - Max 2 retries (configurable)
    - Exponential backoff: 1s, 2s, 4s...
    - Fallback to text extraction if OCR fails
    - Graceful degradation with partial data
    - Complete failure logging
    - Metadata tracking (attempts, method, error details)

RECOVERY FLOW:
  1. Try primary extraction (OCR)
  2. If fails → Wait 1s
  3. Retry OCR
  4. If fails → Wait 2s
  5. Final retry OCR
  6. If fails → Fallback to text extraction
  7. If fallback fails → Graceful degradation
  8. Return degraded data with confidence=0
  9. Log complete recovery attempt

BENEFITS:
  ✓ Transient errors don't fail document
  ✓ Automatic failover to backup method
  ✓ Partial data recovery on complete failure
  ✓ Zero silent failures

ARCHITECTURE & FLOW
================================================================================

PROCESSING PIPELINE (with v2.0 enhancements):

START: process_documents([pdf_list])
  │
  ├─ Batch Processing Loop (SAFE: partial failures OK)
  │  │
  │  └─ For each PDF (independent transaction):
  │     │
  │     ├─ Calculate input_hash(SHA256)
  │     │  └─ File integrity verification
  │     │
  │     ├─ Route Document
  │     │  └─ Determine procurement vs specification
  │     │
  │     ├─ Extract Data
  │     │  ├─ Try primary method
  │     │  ├─ Retry with backoff (max 2)
  │     │  ├─ Fallback to alternate method
  │     │  └─ Graceful degradation (conf=0)
  │     │
  │     ├─ Validate Data
  │     │  ├─ Apply Pydantic schema validation
  │     │  ├─ Check business rules
  │     │  └─ Log any schema errors
  │     │
  │     ├─ Check Confidence Threshold
  │     │  ├─ if confidence < 70%:
  │     │  │  └─ requires_review = True
  │     │  │     status = "REQUIRES_REVIEW"
  │     │  └─ else: Continue
  │     │
  │     ├─ Finalize Output
  │     │  └─ Format for pipeline type
  │     │
  │     ├─ Log Execution (Audit-Grade)
  │     │  ├─ input_hash ...................... File integrity
  │     │  ├─ pipeline_version ............... Version used
  │     │  ├─ extraction_confidence ......... Quality metric
  │     │  ├─ validation_result ............. Pass/Fail/Warning
  │     │  ├─ schema_validation_passed ....... Pydantic result
  │     │  ├─ requires_review ............... Manual review flag
  │     │  ├─ failure_details ............... Stack trace
  │     │  └─ metadata ....................... Complete context
  │     │
  │     └─ Store Result
  │        └─ results[pipeline_type].append(result)
  │        (Exception caught: log + continue)
  │
  ├─ Summary Generation
  │  ├─ Count successful, failed, requires_review
  │  ├─ Track batch processing safety status
  │  └─ Aggregate statistics
  │
  ├─ Save Execution Log
  │  └─ logging/logs/execution_YYYYMMDD_HHMMSS.json
  │     (Complete audit trail with all fields)
  │
  ├─ Save Results
  │  ├─ output/deterministic_procurement_*.json
  │  ├─ output/deterministic_specification_*.json
  │  └─ output/deterministic_pipeline_*.json
  │
  └─ Return Summary
     {
       "batch_processing_safety": {
         "successful": 7,
         "failed": 2,
         "requires_review": 1,
         "total_attempted": 10,
         "partial_failure_note": "..."
       }
     }
